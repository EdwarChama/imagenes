# Empresa de seguros TK-U

## Decripción del Problema 
Dicho programa, solo se pueden hacer las cotizaciones de manera manual, por ende, hay un gran atraso en la entrega de las cotizaciones y se están perdiendo una gran cantidad de posible clientes. 

## Algoritmo para empresa de seguros 
1. Establecer un precio base para todas las cotizaciones.
2. Se le solicita información al asegurado.
3. Se verifica que el asegurado sea mayor de edad, si no cumple con esta condición, no se realiza la cotización. 4.
4. Se verifica que el asegurado esta casado o no, si está casado se le solicita la edad del cónyuge.
5. Se le pide al asegurado responder si tiene hijos, si tiene se le solicita la cantidad.
6. Según la edad del asegurado, se le aplica un recargo según el rango correspondiente. **De 18 a 24 años -> 10% del precio base.** **De 25 a 49 años -> 20% del precio base.** **De 50 años en adelante -> 30% del precio base.**
7. Si el asegurado esta casado/a, se verifica la edad del cónyuge y se le aplica el recargo correspondiente, usando los mismos rangos de edad.
8. Si el asegurado tiene hijos, se le aplica un recargo adicional del **20% del precio base por cada hijo**, sin importar la edad que tengan los hijos.
9. Se suman todos los recargos obtenidos.
10. Se obtiene el precio final de la cotización, el cual cubre al asegurado y a su familia.
11. Se le muestra al asegurado el resultado final obtenido de la cotización.
